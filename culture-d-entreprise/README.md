# Culture — Yoan Maisonneuve

Phrases et maximes signées qui définissent l'identité culturelle transversale de tous mes projets : Cyborg, Askio1, matchllm, cyborg-yoan, Enter-Game, Blueprint, robotique humanoïde, élève IA émancipé, et tous ceux à venir.

## Rôle de ce dossier

- **Transmettre** une vision cohérente à mes futurs collaborateurs, à mes enfants, au lecteur public.
- **Documenter** l'identité éthique, le positionnement, les principes opérationnels qui rendent mes projets reconnaissables.
- **Filtrer** les décisions majeures : toute décision technique, stratégique ou relationnelle doit être compatible avec les maximes signées ici.

## Distinction avec les autres types de fichiers

| Type | Rôle | Audience |
|---|---|---|
| Organes Cyborg (`MODELE.md`, `PERCEPTION.md`...) | Cognition active de l'agent | Claude / agents IA en session |
| Auto-mémoire (`memory/*.md`) | Contexte récurrent inter-sessions | Claude (rappel automatique) |
| **Culture (ce dossier)** | **Identité publique transmissible** | **Collaborateurs, enfants, lecteurs** |
| Pacte (`agents/cyborg/cyborg-yoan/PACTE.md`) | Engagement public daté | Tous + Yoan lui-même |

## Format

- `maximes.md` : collection datée et catégorisée. Chaque maxime a un numéro (`#001`, `#002`...), une catégorie, une origine, et un mode d'application.
- Une maxime est versionnée par date. Si elle est révisée, on garde l'ancienne avec mention *« remplacée par #N le YYYY-MM-DD »*. Pas de réécriture silencieuse.

## Principe fondateur

Une maxime n'est pas une phrase écrite vite et oubliée. **C'est un acte de signature.** Ce qui est ici engage. Ce qui n'est pas ici n'engage pas. Pas de maxime non datée. Pas de maxime non signée.

— Yoan Maisonneuve, 2026-04-29
